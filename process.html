Phase 0

In Phase 0 of my web development project, I delved deeply into analyzing well-designed websites for inspiration, which significantly shaped my design strategy. I hand-drew wireframe sketches, laying out the essential structure and navigation flow of the website. Alongside these sketches, I also crafted an Information Architecture (IA) diagram, ensuring a logical organization of content for optimal user experience. To round out the preparatory work, I meticulously listed all project requirements. This early stage was critical, setting a solid foundation with a clear vision and specific objectives, steering the project's direction from concept to reality.

Phase 1

In Phase 1, I transitioned from the conceptual groundwork laid in Phase 0 to a more concrete visualization of the project. Utilizing the sketches and the Information Architecture (IA) diagram developed earlier, I began the detailed work of transforming these initial ideas into digital wireframes. This process involved carefully applying the IDs and structures outlined in my hand-drawn sketches to create a set of wireframes. These wireframes served as a blueprint for the website, illustrating the placement of elements and the user interaction flow without yet delving into the aesthetic details. Wireframe XD Link

Phase 2

In Phase 2, I advanced from the detailed wireframes created in Phase 1 to the development of interactive prototypes. This phase was characterized by incorporating my own rich media productions, which included images, videos, and interactive elements, to present a more comprehensive look of the envisioned website. These prototypes not only showcased the basic aesthetic and functionality of the site but also served as a practical foundation for the coding phase. By having a tangible, interactive model, I was able to accurately translate the design into code, ensuring that the development phase was closely aligned with the initial design intentions. This approach significantly streamlined the coding process, as it provided a clear and direct reference for implementing the site's layout, features, and style. Prototype XD Link

Coding & WCAG2.0

In the final phase of my web development project, the transition from prototype to production involved coding the design into HTML and CSS. This crucial stage was approached with a keen focus on aesthetics and accessibility. I employed a color picker tool to select high-contrast color sets, ensuring that the website's theme was not only visually appealing but also accessible to all users, including those with visual impairments. The color scheme was carefully chosen to provide sufficient contrast, enhancing readability and user experience.
To ensure the website met accessibility standards, I used the WAVE extension as the video above demostrated, a comprehensive tool for evaluating web accessibility. This allowed me to test the website against the Web Content Accessibility Guidelines (WCAG) 2.0, identifying and rectifying any issues that could hinder access for users with disabilities. To make the website fully accessible, I also chose high-contrast color schemes, aligning with WCAG 2.0 guidelines, to enhance readability for visually impaired users. Successfully passing the contrast ratio test affirmed our adherence to accessibility best practices. Additionally, the WAVE extension's thorough analysis confirmed the site was free from accessibility errors, highlighting a dedicated effort to create an inclusive and universally navigable web experience.
